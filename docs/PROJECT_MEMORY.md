# Nova Quant Project Memory

Last updated: 2026-03-14 (Asia/Shanghai)

## 1) Mission

Nova Quant is an AI-native quant decision platform for US equities and crypto.

Mission:
- reduce emotional trading,
- improve execution discipline,
- provide explainable action plans and risk-aware no-trade decisions.

## 2) Product Positioning

Nova Quant is not:
- an auto-trading black box,
- a signal spam feed,
- a chart terminal clone.

Nova Quant is:
- a structured decision OS with auditable research logic.

## 3) Current Architecture Reality

Core runtime remains in `src/` and is orchestrated by `src/engines/pipeline.js`.

Current canonical product AI path:
- `AiPage` / Ask Nova / Chat sheet now share one backend assistant path via `/api/chat`
- chat threads persist in SQLite (`chat_threads`, `chat_messages`)
- deterministic retrieval is no longer a separate frontend AI brain; it is now a backend fallback/tool
- provider fallback now covers timeout/network/malformed/empty/rate-limit style failures, not only 429

Current engineering reproducibility baseline:
- package scripts now include `clean`, `lint`, `test`, and `verify`
- repo handoff policy now excludes local runtime artifacts through `.gitignore`, `.npmignore`, `.gitattributes`, and `package:source`

Research core now includes:
- strategy family registry
- strategy discovery engine (hypothesis -> template -> candidate -> validation -> scoring -> promotion)
- research materials foundation (official-source digest, universe seeds, hypothesis/template/question libraries, market playbooks, weekly feed template)
- advanced research knowledge pack (methodology doctrine, portfolio intelligence standards, validation doctrine, failure-mode taxonomy, governance checklist standards, weekly/monthly research ops)
- research evidence system (lineage and promotion-history traceability)
- portfolio simulation engine (multi-strategy, multi-asset, risk-budget-aware)
- AI research copilot (diagnostics-driven research suggestions)
- weekly research cycle and report generation
- feature/signal lifecycle abstraction
- regime system with confidence + transitions
- layered risk budget objects
- funnel diagnostics with threshold sensitivity
- shadow opportunity analysis
- walk-forward anti-overfit validation
- lifecycle governance operations
- research automation loop

Primary module entry:
- `src/research/core/researchCoreUpgrade.js`

## 6) Known Weak Areas

1. Provider orchestration is more resilient, but assistant tool execution is still prompt-driven rather than native structured function calling.
2. Backtest realism and paper/backtest consistency still need deeper correction in some older research modules.
3. Shadow forward path remains partly synthetic where replay coverage is missing.
4. Cost/slippage models remain simplified (venue-specific calibration pending).
5. Governance reviewer workflow is still system-generated by default.
6. Portfolio simulation currently mixes stronger canonical replay paths with older proxy-style paths in some secondary diagnostics.
7. Weekly report history persistence and reviewer sign-off workflow remain lightweight.
8. Advanced pack policies are documented and seeded, but runtime policy-enforcement wiring is still partial.

## 7) Non-Negotiables

- Auditability over black-box outputs.
- OOS realism over in-sample vanity.
- Risk boundary clarity over aggressive signal density.
- Product simplicity with evidence-backed reasoning.
- Every major implementation block must update project memory and logs per `docs/SESSION_HANDOFF_PROTOCOL.md`.
